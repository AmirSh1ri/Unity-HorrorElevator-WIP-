# Elevator Horror Elevator game - WIP

## Description
A first-person Unity horror game built around a **camera-based visibility mechanic**. Enemies are **not visible by default** and only revealed when directly viewed through a **rechargeable in-game camera**, creating tension through limited visibility and resource management.

The project focuses on **camera systems, visibility checks, and controlled enemy reveal logic**, using player perspective as a core gameplay mechanic. The game is structured with future publishing and release in mind.

## Key Systems Implemented
- Custom visibility detection system (camera-based rendering logic)
- Multiple different enemy AI and behaviours.
- Line-of-sight checks using raycasting for enemy AI.
- Resource management system (battery / recharge).
- Unity tools created for better testing and management.
- Save system (WIP).

## Collaboration
- Core gameplay systems, mechanics, and tools were developed from scratch without relying on third-party scripts.
- 3D assets and animations were primarily created using Blender to support a fully custom pipeline.
- The project emphasizes building foundational systems and understanding underlying implementations rather than relying on pre-built solutions.

- 21 uniquely designed rooms with varied objectives (implemented).
- Managing different resources (4 types of resources).
- Different mini games, puzzles and a genre of horror in each floor.
